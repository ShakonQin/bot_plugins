# NcatBot 插件使用说明

本目录包含三个社区插件，均基于 NcatBot 插件规范开发，支持 BoChat 平台与 QQ 的互通。

---

## 1. bochat_forwarder — BoChat ↔ QQ 双向消息转发

### 功能

在 BoChat 群聊与 QQ 群聊/私聊之间双向转发消息，支持多条路由规则、关键词过滤和自定义格式。

### 配置 (`config.yaml`)

```yaml
bochat:
  base_url: "http://127.0.0.1:8080"
  account: "test_user_01"
  password: "Test1234!"
  bot_id: ""  # 留空则自动选择第一个 Bot

routes:
  - name: "bochat群转QQ群-示例"
    direction: "bochat_to_qq"       # bochat_to_qq | qq_to_bochat
    bochat_group_id: "g_xxx"
    qq_group_id: "130445095"
    enabled: true
    filter:
      ignore_bots: false            # 是否忽略 Bot 消息
      keywords: []                  # 关键词白名单（空=全部转发）
    format:
      show_sender: true
      prefix: "[BoChat|{sender}] "  # 支持 {sender}、{group} 变量
```

### 消息处理

| 方向 | 行为 |
|------|------|
| BoChat → QQ | 提取 `content.text`，加上发送者前缀后发送到 QQ 群 |
| QQ → BoChat | 去除 CQ 码，将 @、图片、表情等转为文本占位符后发送到 BoChat 群 |

### 防循环

- 自动忽略自身 Bot 发出的消息
- 可通过 `filter.ignore_bots` 忽略所有 Bot 消息

## 2. translator — 中英互译

### 功能

通过 `/trans` 命令实现中英文互译，同时支持 BoChat 群聊和 QQ 群聊/私聊。

### 命令

```
/trans <模式> <文本>

模式:
  c2e  中文 → 英文
  e2c  英文 → 中文

示例:
  /trans c2e 你好世界
  /trans e2c Hello World
```

发送 `/trans`（不带参数）会返回用法提示。

### 配置 (`config.yaml`)

```yaml
bochat:
  base_url: "http://127.0.0.1:8080"
  account: "test_user_01"
  password: "Test1234!"
  bot_id: ""

translate:
  provider: "mymemory"   # mymemory（免费）| baidu（需申请密钥）

  mymemory:
    email: ""            # 可选，填写后提高请求配额

  baidu:
    app_id: ""
    secret_key: ""
    # 申请地址: https://fanyi-api.baidu.com/
```

### 翻译后端

| 后端 | 特点 |
|------|------|
| MyMemory | 免费，无需注册，适合轻量使用 |
| Baidu | 需注册，每月 5 万字符免费额度，翻译质量更高 |

### 回复格式

```
[中→英] Hello World
[英→中] 你好世界
```

## 3. github_subscriber — GitHub 事件订阅

### 功能

轮询 GitHub Events API，将仓库动态推送到 QQ 群和/或 BoChat 群。支持事件类型过滤和去重。

### 配置 (`config.yaml`)

```yaml
github:
  token: ""              # 可选，填写后 API 速率限制从 60→5000 次/小时
  poll_interval: 60      # 轮询间隔（秒），最小 30

bochat:
  base_url: "http://127.0.0.1:8080"
  account: "test_user_01"
  password: "Test1234!"
  bot_id: ""

subscriptions:
  - repo: "owner/repo-name"
    enabled: true
    events:              # 要监听的事件类型
      - push
      - issues
      - pull_request
      - release
      - star
    targets:
      qq_groups:
        - "130445095"
      bochat_groups:
        - "g_xxx"
```

### 支持的事件类型

| 事件 | 说明 |
|------|------|
| `push` | 代码推送 |
| `issues` | Issue 创建/关闭/评论 |
| `pull_request` | PR 创建/合并/关闭 |
| `release` | 新版本发布 |
| `star` | Star/Unstar |
| `fork` | 仓库被 Fork |
| `issue_comment` | Issue 评论 |
| `create` | 分支/标签创建 |
| `delete` | 分支/标签删除 |

### 推送格式示例

```
[GitHub] owner/repo
📌 Push: user 推送了 3 个提交到 main
🔗 https://github.com/owner/repo/compare/abc...def
```

### 高效轮询

- 使用 ETag 机制，无新事件时 GitHub 返回 304 不计入速率限制
- 基于 `last_event_id` 去重，不会重复推送

## 通用说明

### 依赖安装

三个插件均需要以下 pip 依赖（插件加载时会自动提示安装）：

```
bochat-sdk[ws]>=0.1
httpx>=0.27,<1.0
pyyaml>=6.0
```

### BoChat 连接

所有插件通过 `BochatBridge` 连接 BoChat 平台：
1. 使用 account/password 登录获取 user_token
2. 选择或创建 Bot，获取 bot_token
3. 建立 WebSocket 连接接收实时消息
4. 通过 HTTP API 发送消息

`bot_id` 留空时自动选择账户下第一个可用 Bot。

### 插件加载

将插件目录放置在 NcatBot 的 `plugins/` 目录下，启动时自动扫描加载。支持热重载——修改插件文件后自动重新加载。
